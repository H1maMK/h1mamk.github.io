# Миграция данных из MySQL в MongoDB

Этот документ описывает процесс миграции данных из существующей MySQL базы данных в MongoDB для React приложения DeviceMaster.

## Подготовка к миграции

### 1. Установка зависимостей

```bash
cd devicemaster-react/backend
npm install mysql2
```

### 2. Настройка подключения к MySQL

Отредактируйте файл `scripts/migrateFromMySQL.js` и укажите правильные параметры подключения к MySQL:

```javascript
const mysqlConfig = {
  host: 'localhost',        // Хост MySQL сервера
  user: 'root',            // Имя пользователя MySQL
  password: 'your_password', // Пароль MySQL
  database: 'praktik',     // Имя базы данных
  charset: 'utf8mb4'
};
```

### 3. Проверка MongoDB подключения

Убедитесь, что MongoDB запущен и доступен. Проверьте переменную окружения `MONGODB_URI` в файле `.env`:

```
MONGODB_URI=mongodb://localhost:27017/devicemaster
```

## Запуск миграции

### Полная миграция

Для запуска полной миграции всех данных:

```bash
npm run migrate:mysql
```

Или напрямую:

```bash
node scripts/migrateFromMySQL.js
```

### Что мигрируется

Скрипт мигрирует следующие данные в указанном порядке:

1. **Категории товаров** (`categories` → `Category`)
   - Название категории
   - Тип устройства (Игровое/Офисное)

2. **Пользователи** (`users` + `roles` → `User`)
   - Имя пользователя, email, пароль (хешированный)
   - Роль (admin/user)
   - Профиль (год рождения, пол, аватар)

3. **Товары** (`products` → `Product`)
   - Название, описание, цена
   - Изображения (до 3 штук)
   - Характеристики (парсятся из поля properties)
   - Связь с категорией
   - Количество на складе

4. **Статьи** (`articles` → `Article`)
   - Заголовок, содержание
   - Изображение статьи
   - Дата публикации

5. **Заказы** (`orders` + `order_items` → `Order`)
   - Связь с пользователем
   - Товары в заказе с количеством и ценой
   - Общая сумма заказа
   - Статус заказа

6. **Отзывы** (`reviews` → встроенные в `Product`)
   - Рейтинг и комментарий
   - Связь с пользователем
   - Дата создания

## Особенности миграции

### Сохранение связей

Скрипт сохраняет оригинальные ID из MySQL в поле `mysqlId` для возможности повторной миграции и отладки.

### Обработка дубликатов

При повторном запуске скрипт проверяет существование записей и пропускает дубликаты:
- Категории проверяются по названию
- Пользователи по username и email
- Товары по названию
- Статьи по заголовку
- Заказы по mysqlId

### Парсинг характеристик товаров

Характеристики из поля `properties` в MySQL парсятся в объект:

**MySQL:**
```
Цвет: черный; Тип: беспроводная; DPI: до 25600
```

**MongoDB:**
```json
{
  "Цвет": "черный",
  "Тип": "беспроводная", 
  "DPI": "до 25600"
}
```

### Хеширование паролей

Все пароли пользователей хешируются с помощью bcrypt. По умолчанию используется пароль 'password' для всех пользователей.

## Проверка результатов

После миграции скрипт выводит статистику:

```
📊 Статистика миграции:
- Категории: 12
- Пользователи: 1
- Товары: 12
- Статьи: 4
- Заказы: 0
```

### Проверка в MongoDB

Вы можете проверить данные в MongoDB:

```bash
# Подключение к MongoDB
mongo devicemaster

# Проверка коллекций
db.categories.count()
db.users.count()
db.products.count()
db.articles.count()
db.orders.count()

# Просмотр примера товара
db.products.findOne()
```

## Устранение проблем

### Ошибка подключения к MySQL

```
❌ Ошибка подключения к MySQL: Error: connect ECONNREFUSED
```

**Решение:** Проверьте, что MySQL сервер запущен и параметры подключения корректны.

### Ошибка подключения к MongoDB

```
❌ Ошибка подключения к MongoDB: MongoNetworkError
```

**Решение:** Убедитесь, что MongoDB запущен и доступен по указанному адресу.

### Дублирование данных

Если нужно очистить MongoDB перед повторной миграцией:

```bash
npm run db:reset
```

## Дополнительные команды

```bash
# Проверка подключения к базам данных
npm run db:check

# Инициализация MongoDB (создание индексов)
npm run db:init

# Очистка MongoDB
npm run db:reset
```

## Примечания

- Миграция сохраняет все изображения как URL-ссылки
- Отзывы встраиваются в документы товаров
- Заказы создаются только если найдены соответствующие пользователи и товары
- Все даты сохраняются в оригинальном формате
